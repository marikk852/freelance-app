## Выявленные уязвимости и меры защиты

### ✅ Закрыто
1. **Несанкционированный release** — двойная проверка: 1) delivery.status='approved' в БД, 2) статус на блокчейне
2. **Double-spend** — проверка статуса escrow перед любой операцией с деньгами
3. **SQL Injection** — везде параметризованные запросы `$1, $2`
4. **Replay Attack на Telegram** — проверка auth_date (не старше 24ч) + HMAC-SHA256
5. **Подмена user_id** — verifyTelegramInitData пересчитывает hash
6. **Утечка оригинальных файлов** — AES-256-CBC + storage/encrypted/ недоступен напрямую
7. **Утечка данных в превью** — код 35%, Excel `***`, PDF только стр.1
8. **Захардкоженные ключи** — статический анализ файлов сервисов
9. **DoS** — rate limiting 100 req/15min через express-rate-limit
10. **XSS** — helmet() middleware устанавливает security headers

### ⚠️ Рекомендации (вне MVP)
- Добавить аудит доступа к файлам (кто и когда скачал)
- Ротация ENCRYPTION_KEY с перешифровкой старых файлов
- 2FA для арбитра при вызове release/refund
- Мониторинг аномалий (много release за короткое время)
- Аудит смарт-контракта сторонней компанией перед снятием лимита $500
