## Реализованные тесты

### contracts/escrow.spec.ts (325 строк, @ton/sandbox)
Тестирует FunC смарт-контракт в изолированной среде без реального блокчейна.

Покрытые сценарии:
- Нормальное закрытие: deposit → release → проверка баланса фрилансера
- Возврат: deposit → refund → проверка баланса клиента
- Спор с split: deposit → split(60) → 60% фрилансеру, 40% клиенту
- Таймаут дедлайна: авто-refund после истечения срока
- Безопасность: попытка release не арбитром → отклонение
- Безопасность: двойной release → отклонение

## Известные уязвимости (УСТРАНЕНЫ в коде)
1. shape validation min:1 — предотвращает пустую строку
2. throttle:5,1 на /order — rate limiting
3. images array max:10 — ограничение загрузок
4. UUID validation на fileId — предотвращает path traversal
5. timingSafeEqual для admin пароля — предотвращает timing attacks
6. Participant check на каждом endpoint — предотвращает несанкционированный доступ
